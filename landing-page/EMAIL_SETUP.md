# メール送信設定ガイド

## Gmailアプリパスワードの設定方法

予約フォームからメールを送信するには、Gmailのアプリパスワードが必要です。

### 手順

1. **Googleアカウントにアクセス**
   - https://myaccount.google.com/ にアクセス
   - `raito.5963.co@gmail.com` でログイン

2. **2段階認証を有効化**（まだの場合）
   - 左メニューから「セキュリティ」を選択
   - 「Googleへのログイン」セクションで「2段階認証プロセス」をクリック
   - 画面の指示に従って設定

3. **アプリパスワードを生成**
   - 「セキュリティ」ページに戻る
   - 「Googleへのログイン」セクションで「アプリパスワード」を探す
   - 「アプリパスワード」をクリック
   - アプリを選択: 「メール」
   - デバイスを選択: 「その他（カスタム名）」
   - 名前を入力: 「Modulate Landing Page」
   - 「生成」をクリック
   - **16文字のパスワードが表示されます**（スペースなし）

4. **.env.localファイルを更新**
   ```bash
   EMAIL_USER=raito.5963.co@gmail.com
   EMAIL_PASS=生成された16文字のアプリパスワード
   ```

5. **開発サーバーを再起動**
   ```bash
   npm run dev
   ```

## 注意事項

- アプリパスワードは一度しか表示されないので、必ずコピーして保存してください
- スペースは含めず、16文字の英数字をそのまま使用してください
- `.env.local`ファイルは`.gitignore`に含まれているため、Gitにはコミットされません
- アプリパスワードは通常のパスワードとは異なり、このアプリ専用のものです

## トラブルシューティング

### エラー: "Invalid login: 534-5.7.9 Application-specific password required"
→ 2段階認証が有効で、通常のパスワードを使用している場合に発生します。上記の手順でアプリパスワードを生成してください。

### エラー: "Invalid login: 535-5.7.8 Username and Password not accepted"
→ メールアドレスまたはパスワードが間違っています。`.env.local`の設定を確認してください。

### メールが送信されない
1. `.env.local`ファイルが正しい場所にあるか確認
2. 開発サーバーを再起動
3. ブラウザのコンソールでエラーを確認

## 代替案：他のメールサービスを使用

Gmailの代わりに他のSMTPサービスを使用することもできます：

### SendGrid（推奨）
- 無料プランで1日100通まで送信可能
- https://sendgrid.com/

### Mailgun
- 無料プランで月5,000通まで
- https://www.mailgun.com/

### Resend
- 無料プランで月3,000通まで
- https://resend.com/

これらのサービスを使用する場合は、`app/api/preorder/route.ts`のSMTP設定を変更する必要があります。
